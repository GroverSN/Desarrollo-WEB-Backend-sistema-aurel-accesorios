# PROTOCOLO DE DESARROLLO: SISTEMA DE AUREL ACCESORIOS (ACADÉMICO/EMPRENDIMIENTO)

## 🎯 Objetivo del Proyecto
Desarrollar un sistema de gestión de inventario y catálogo de joyas con Node.js (NestJS) y Vue.js, cumpliendo con estándares de seguridad, reportes y despliegue en contenedores.

## 🛠️ Stack Tecnológico (Estricto)
- **Backend:** NestJS con TypeScript.
- **Base de Datos:** PostgreSQL con TypeORM.
- **Gestor de Paquetes:** pnpm.
- **Pruebas de API:** Thunder Client.
- **Frontend:** Vue.js + Vite.

## ⌨️ Reglas de Codificación (Estilo del Docente)
- **Tipados:** No usar la sintaxis `Array<Tipo>`, usar siempre `Tipo[]`.
- **Funciones:** Preferencia por Arrow Functions (`const miFuncion = async () => {}`).
- **Asincronía:** Uso obligatorio de `async/await`.
- **Estructuras:** Uso de Interfaces en lugar de Types donde sea posible para definiciones de objetos.
- **Idioma:** Código (variables, clases, métodos) en inglés; comentarios y logs en español si es necesario.

## 🚀 Requisitos de Lógica de Negocio (Rúbrica)
1. **Seguridad:** - Autenticación con JWT.
   - Encriptación de contraseñas (Bcrypt).
   - Validador de complejidad de contraseña (Débil, Intermedia, Fuerte).
   - Implementación de CAPTCHA para el login.
2. **Log de Accesos:** - Tabla `logs` que registre: usuario_id, ip, evento (LOGIN/LOGOUT), browser, fecha y hora.
3. **Persistencia:**
   - CRUDs con **Eliminación Lógica** (Soft Delete usando la columna `deletedAt`).
   - Validaciones de entrada mediante DTOs (class-validator).
4. **Reportes y Gráficos:**
   - Generación de reportes en PDF.
   - Endpoints preparados para gráficos estadísticos (ventas por mes, productos más vendidos).
5. **DevOps:**
   - Configuración de Dockerfile y Docker Compose.
   - Manifiestos de Kubernetes (K8s).

## 🗄️ Arquitectura de Base de Datos (Tablas Definidas)
El agente debe seguir este esquema relacional:
- **usuarios:** id, nombre, email, password, rol, status, deletedAt.
- **productos:** id, codigo, nombre, descripcion, precio, stock, categoriaId, imagenId, status, deletedAt.
- **categorias:** id, nombre, descripcion, status, deletedAt.
- **imagenes:** id, url, public_id (para almacenamiento en la nube o local).
- **ventas:** id, fecha, usuarioId, total, metodoPago.
- **detalle_ventas:** id, ventaId, productoId, cantidad, precioUnitario, subtotal.
- **logs_acceso:** id, usuarioId, ip, evento, browser, timestamp.
